# Agent Spec: ambient-expense-agent

## Overview
The `ambient-expense-agent` is an event-driven, scheduled background agent designed to track, process, and log expenses from ambient data sources, such as emails (via Gmail) and files (via Google Drive). It runs non-interactively, processing incoming notifications and documents, extracting expense details, and maintaining a ledger or reporting alerts.

## Example Use Cases
- **Email Receipt Processing**: The agent scans the inbox on a schedule or reacts to notifications for messages matching "receipt", "invoice", or "order confirmation". It parses the PDF or inline text to extract transaction date, vendor, amount, and category, and saves the data.
- **Drive Folder Processing**: The agent monitors a Google Drive folder for newly uploaded expense documents, scans them, and categorizes the expense items.

## Tools Required
- **Gmail Tool**: Connects to the Gmail API to search, read, and mark emails as processed.
- **Google Drive Tool**: Connects to the Google Drive API to retrieve files (PDFs, images) for scanning.
- **Expense Extraction Tool**: Uses LLM-based structured extraction to parse expense details from unstructured text or documents.

## Constraints & Safety Rules
- **No Deletions**: The agent must never delete emails or files from Gmail or Google Drive. It should only mark them with labels or move them to a processed folder.
- **Read-Only Scopes**: Use the most restrictive OAuth scopes possible (e.g., read-only for reading emails and documents).
- **No Financial Approvals**: The agent must only record/log expenses and never make or approve payments.

## Success Criteria
- Successfully connects to Gmail and Drive APIs to retrieve sample receipt emails and documents.
- Accurately parses transaction details (amount, currency, vendor, date) from unstructured sources.
- Deploys reliably to Google Kubernetes Engine (GKE Autopilot) with basic operational logging.
